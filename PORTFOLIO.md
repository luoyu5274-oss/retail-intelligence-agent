# Retail Intelligence Agent — 运动品牌竞品分析平台

## 一句话描述

全栈 AI 应用：自动解析 Nike/Adidas/Lululemon 年报 PDF，LLM 提取财务数据，向量检索 + 智能问答，支持跨品牌日历对齐对比。

## 技术栈

**Backend:** Python · FastAPI · pdfplumber · scikit-learn (TF-IDF) · DeepSeek API (V4 Flash) · SSE 流式推送
**Frontend:** React 19 · Recharts 2 · Vite 8 · CSS Variables 主题系统
**数据:** 静态 JSON 数据层 · 向量索引 · 季度日历对齐映射

## 核心功能

- **智能问答**：基于 12 份年报 PDF（~5000 页）的 TF-IDF 向量检索 + LLM 上下文增强，支持跨品牌对比分析。回复中表格可一键切换为内嵌图表（柱状图/折线图/组合图）
- **财年日历对齐**：解决 Nike(5月)/Adidas(12月)/Lululemon(1月) 财年错位问题，季报按实际月份映射到统一日历轴
- **仪表盘**：KPI 卡片、收入/利润率/增速图表、品牌竞争力雷达、区域营收分布
- **无需初始化**：仪表盘数据采用静态 JSON 架构，页面秒开，不依赖 LLM 调用

## 架构亮点

| 决策 | 方案 | 原因 |
|------|------|------|
| 向量检索 | TF-IDF 替代 SentenceTransformer | CPU 环境下索引时间从 16 分钟降至 2 秒 |
| 仪表盘数据 | 静态 JSON 替代 LLM 实时提取 | 消除提取质量不稳定 + 15 分钟等待 |
| 财务提取 | 5 步流水线（提取→归一化→校验→重试→反向计算） | 自动修复 LLM 输出错误（单位、缺失字段） |
| 前端状态 | 仪表盘 ready 与问答 ready 分离 | 仪表盘无需索引即可使用 |
| 季报映射 | 按实际月份而非名义季度标签映射 | Nike"Q1"=6-8月 → 日历 Q3，而非直接对比 Adidas Q1 |

## 数据规模

- 12 份年报 PDF（年均 300+ 页，共 ~5000 页）
- 9 份季报 PDF
- 2,450 个文本检索块
- 3 品牌 × 4 财年 × 20+ 财务指标
- 9 个季度数据点（日历对齐）
